## 2. 김동현

```
### 프로젝트에서 좋았던 점, 배운점 :

- 프로젝트 문서화
 기획단계에서부터 회고까지 문서화를 잘 진행해서 너무 좋았습니다.
특히 기획 단계에서 API 및 기능 명세서를 Notion에 문서화하고 이를 기반으로 서로 소통 및 개발을 진행하다보니,
불필요한 중복적인 질문이나 소통이 사라지고 효율적인 소통이 가능했습니다.
그리고 개발을 진행하면서 기획이나 기술적인 부분에 쟁점이 발생했을 때도,
초기의 기획의도와 기술 선정 이유들을 쉽게 복기하면서 빠른 판단을 내릴 수 있었습니다.

- AI를 통한 자연어 처리 경험
 이전까지 머신러닝/딥러닝 이론적인 부분의 공부 외에는 실질적으로 프로젝트를 진행하면서 기술을 도입해본 경험이 없었습니다.
이번 프로젝트를 통해서 자연어처리 기술의 요즘 트랜드가 어떠한지, 그리고 pipeline을 어떻게 구성하고,
프로젝트에 자연어처리 기술을 도입할 때 어떤 것들을 고려해야하는지 알 수 있었습니다.

### 프로젝트에서 아쉬웠던점 :

- NLU 고도화
 RASA로 사용자의 발화 의도를 파악하는 기능을 구현하면서 예상치 못한 부분에서 에러가 발생했습니다.
학습시킨 질문과 전혀 다른 의도의 질문을 입력하면 예외 처리 intent로 판단하도록 학습시켰습니다.
하지만, 일부 질문은 원하는대로 잘 분류 되었지만, 일부는 예외처리 되지 않고 전혀 다른 intent 로 출력되기도 했습니다.

해당 문제가 왜 발생하는지 정확하게 파악하지 못해서 많이 답답했습니다.
결국 학습키는 intent들을 더 세분화하고, 학습시키는 질문의 양을 늘려서 일부 개선해냈습니다.
하지만 완벽하게 개선해냈다고 자신할 수 없어서 추후 고도화를 하면서 성능을 더 개선해보고 싶습니다.
```

## 3. 이민하

```
프로젝트 주제가 이전에 했었던 프로젝트들 보다 흥미로워서 기대를 많이 했지만 기대 속에 걱정도 있었다.
적용하려는 기술의 난이도나 시간적인 면을 고려 했을 때 높은 퀄리티를 내기는 힘들지 않을까 생각했기 때문이다.
하지만 초엘리트들만 모인 우리팀.. 예상했던 시간보다 훨씬 빠르게 기능 구현을 끝내버렸다😮
팀원들이 각자 맡은 역할을 충실히 해준 덕분에 컨설턴트님과 코치님 싸피 동기들 심지어는 도메인 전문가님까지..
모두에게 수많은 칭찬을 들을 수 있었던거 같다.

개인적으로도 성장한 점이 많았던 프로젝트이다.
처음으로 팀장이라는 직책을 맡으며 잘할 수 있을까 걱정되었지만 이왕 맡은거 열심히 해보자고 다짐하며
노션 정리도 열심히하고, 지라도 어떻게 해야 팀원들이 더 편하게 써서 개발에 좀 더 집중할 수 있을 지 이런저런 시도를 하며 고민해봤다.
귀찮고 머리 아플 때도 많았지만 이런 저런 생각을 하며 확실히 전보다 더 나은 내가 될 수 있었다.
그리고 개발을 할 때도 클린 아키텍처, Hilt, MLKit, 멀티파트 등 사용해보고 싶었던 기술들을 사용할 수 있어 좋았다.
그래도 아직 어색한 부분이 있어 조금 더 공부를 해봐야 할 거 같다. 자율 프로젝트 때 시간 나면 공부 해봐야지..ㅎㅎ
또 같은 스택을 개발하는 팀원이 3명이나 있는 것은 이번 프로젝트가 처음이라 서로 막히는 것이 있을 때 같이 고민하고 해결한 경험도 좋았다.
이번 프로젝트를 하면서 협업의 묘미를 제대로 느낄 수 있었다.
```

## 5. 조수연

```kotlin
### 좋았던 점
협업 툴 Jira를 효율적으로 잘 활용한 것 같아서 좋았다. 팀장이 협업 진행 룰이나 일정 관리를 편하게
할 수 있도록 환경을 만들어주어서 프로젝트 기간 동안 일정에 차질 없이 진행되어서 좋았다.
각 포지션에 해당하는 팀원들간의 코딩 컨벤션이나 규칙들을 정하고 프로젝트를 시작해서 좋았다.
개발 진행 도중 큰 이슈없이 잘 마무리 되어서 좋았고, 클린아키텍처, Hilt 의존성 주입과 같은 안드로이드
지식 부문에서 조금 더 깊게 공부해 볼 수 있어서 좋았다.

### 문제를 고민하거나 다음에 해보고 싶은 것
1. STT
구글 STT 음성 엔진을 사용해서 음성 인식 기능을 구현하였는데, 기획 단계에서 생각했던 부분은
음성인식 엔진이 계속 실행되어서 특정 단어가 인식되면 그 단어 기준으로 뒤에 명령을 서버에 요청
하는 방식으로 기획하였는데, 안드로이드 SpeechRecognizer 클래스에서는 해당 기능을 지원하지 않아
음성인식이 한번 끝나면 다시 시작하도록 코드를 구현해서 해당 부분을 구현하였는데, 다음에는 조금 더
공부해보고 음성인식 엔진이 계속 실행되도록 구현해보고 싶다.

2. Lifecycle Observer
이번 프로젝트에서는 ViewModel에서 데이터는 LiveData로 처리하였는데, 뷰에 해당하는 Fragment에서
옵저버를 호출하면 옵저버를 따로 제거해주지 않으면 해당 프래그먼트가 다시 호출되었을 때 옵저버가
중복으로 생성되어서 에러가 발생하는 이슈가 있었다. 해당 부분을 LiveData을 상속하는 SingleLiveEvent
로 처리하여서 해당 옵저버가 호출되어서 관찰하는 동안 LiveData 값이 변경되면 해당 로직을 처리하고 옵저버를
제거하도록 구현하여서 해결하였는데, 다음에 기회가 된다면 Coroutine Flow를 공부해서 비동기 스트림을 통해
데이터를 전달하는 방식을 구현해보고 싶다.

3. Architecture
이번 프로젝트에서는 SAA(Single Activity Architectrue), Clean Architecture, MVVM 패턴 아키텍처로
프로젝트를 설계하였는데, 확실히 클린 아키텍처를 사용해서 관심사를 분리하여 프로젝트를 개발하니 유지보수 부분이나
재사용성 부분에서 개발하기 좋았다. 하지만 MVVM 패턴에서 적용하다 보니 View와 ViewModel 간의
의존성이 커졌는데, 다음에는 MVVM 패턴과 MVI 패턴을 적용해서 View와 ViewModel 사이의 Intent를
통해 의존성 문제를 해결해보고 싶다.
```

   - 주변 사람들에게 내가 공부한 지식을 공유할 수 있었다.
2. 낯선 기술이라도 포기하지않고 DOCS, 블로그 등을 참고해가며 공부했다.
   - Jenkins를 통한 CI/CD 구축을 통해 팀의 개발 생산성을 높일 수 있었고,
   - 로그 중앙화를 통해 팀원들 모두가 간편하게 로그를 확인할 수 있는 환경을 만들 수 있었다.
3. 개발 관련 서적을 통해 다양한 기술에 대해 간접 경험을 해볼 수 있었다.
   - 우연히 fluentd 로깅 툴에 대해 알게 되었고, 관련 툴을 검색해보다가 우리 프로젝트에 적용하는 계기가 되었다.

   - 확장성 등 여러가지 기준을 적용하다보면 최선의 방법을 찾을 수 있다고 생각한다.
   - 여러가지 기준에 대해 깊게 생각해보는 습관을 가져봐야겠다.

	- 서버가 악의적으로 탈취 되었을 경우 보안에 취약하다고 생각한다. 조금 더 높은 수준의 보안 환경을 구축해보고 싶다.
